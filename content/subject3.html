Object Oriented Programming - Project Analysis

 I'd like to take an special look into one of the exercises: In the last from the 6th list, i managed to make an turn-based text rpg, where i could put in proof what i've been studing, like repeating loops, decision estructures, oop and even frames.

Here we can see the classes i created to this game. Turning the player and boss into objects granted me an easier way to manipulate it atributes, the "parryWindow" is also an object, but we gonna se it more further.

 By now, heres the full explained

The main starts normally, importing libraries and instancing the player and others objects, the player class requires signatures since you choose your name, the other stats are auto filled

How the history is like you're in the end of an adventure, i used the random class to pick in an range to health, strength and defense, to simulate you've passed a lot of battles

The story is told in the console, certain points is required a response of t he player to progress, empty scanners serves to prevent the code to continue. That while loop exists to prevent the story to continue until the player agrees with the condition he is told

The battle consists in turns, so the principle of the code is: your move, the boss's move and then an function to analyse the state of the battle: if you died, if the boss died, etc

Now and further in this code, i utilize the outerloop system to be able to choose when i want to simply restart an loop or if i want to break and continue with the code

How the turns are based in pre determinated choises, to both opponents, i turned them into functions, by doing so i can keep the loop clean and functional

A switch case interprets how the battle is going in basis of a number the checkBattleState() return to it
